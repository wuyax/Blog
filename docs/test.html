# macOS排除指定文件或文件夹同步到iCloud

iCloud同步文件很方便，但是它不能排除指定文件和文件夹的同步。我们很希望有一个像.icloudignore 一样的配置文件来配置那些不需要同步的文件，比如node_modules就是我第一个像屏蔽掉的。遗憾的是并没有这样的配置文件，Apple也没有提供配置工具。

好在有两种文件是不会被同步的，.tmp 和 .nosync。也就是说你只要把文件/文件夹的后缀改为.tmp或.nosync文件/文件夹就不会被同步了。这样修改以后文件夹可以正常使用，但是文件在打开的时候就出问题了，因此你还可以把后缀改为中缀，例如test.pdf.nosync改为test.nosync.pdf 这样，文件能正常打开还不会上传到iCloud。 如果有两个或者多个后缀的文件（xxxx.tar.gz）只要把.nosync插入到文件名后面就可以了（xxxx.nosync.tar.gz）

如果你觉得一个一个改名字太麻烦了，可以使用一些自动化工具来批量重命名。

我觉得最简单的还是建一个文件夹（myself.nosycn），把不需要同步的文件全放里面。

最后再说一下node_modules文件夹，这个文件夹我们不能或者说不建议去改它的名字，因为依赖关系会被破坏，那么要怎么把这个_恶心_的文件夹排除在外呢？

在Twitter上一位网友给出了一个解决方案： mkdir .node_modules.nosync && ln -s .node_modules.nosync node_modules 这个方案的思路比较有意思：先创建一个不会被同步的隐藏文件.node_modules.nosync 然后再给该文件创建一个软连接，也就是说文件实际上保存在.node_modules.nosync文件夹里（不会被同步），当你通过目标文件夹（node_modules）访问文件的时候，实际访问的是.node_modules.nosync文件夹里的文件。

Twitter网友提供的方法经过我的测试是可行的。只是有一点需要注意的：文件夹中已经存在node_modules 是无法用上面的命令实现的，因为这个时候的node_modules不是一个快捷方式而是一个实实在在的文件夹，所有的文件都会上传的iCloud。

综上: 1.最好的办法是创建一个.nosync 的文件夹把不需要同步的文件放里面；2.如果文件夹的名字不可以更改，那就通过创建软连接的方式，把真实文件放在一个.nosync 文件夹里。
